# خطة تطوير منتج شيّرلي (Shairley) — الخطة الشاملة

> **الهدف:** تحويل شيّرلي من أداة مشاركة ملفات إلى منصة تعاون ملفات متعددة المسارات، مع الحفاظ على هوية المنتج الموحدة وعدم تشتيت التجربة.

---

## 1. فلسفة المنتج — القاعدة الذهبية

```
┌─────────────────────────────────────────────────────────────┐
│                    شيّرلي = نواة واحدة                        │
│                                                             │
│   "منصة تعاون ملفات ذكية — مكان واحد لكل ملف مهم"          │
│                                                             │
│   ┌───────────┐   ┌───────────┐   ┌───────────────┐        │
│   │  أكاديمي  │   │  فرق عمل  │   │ مشاركة آمنة  │        │
│   │  (Edu)    │   │  (Teams)  │   │  (Secure)    │        │
│   └─────┬─────┘   └─────┬─────┘   └──────┬────────┘        │
│         │               │                │                  │
│         └───────────────┼────────────────┘                  │
│                         ▼                                   │
│              ┌─────────────────────┐                        │
│              │    النواة المشتركة   │                        │
│              │  (Shared Core)      │                        │
│              └─────────────────────┘                        │
└─────────────────────────────────────────────────────────────┘
```

### المبدأ الأساسي:
- **النواة واحدة** — نفس نظام المساحات، الملفات، الأعضاء، الإشعارات
- **المسارات طبقات** — كل مسار يُفعّل ميزات إضافية فوق النواة
- **المستخدم يختار** — عند إنشاء مساحة عمل، يختار نوعها (أكاديمي / فريق / آمن)
- **لا تكرار** — الميزة تُبنى مرة واحدة وتُفعّل حسب السياق

## 3. المسار الأكاديمي (Edu Track) 📚

### الجمهور المستهدف:
- طلاب جامعات ومجموعات دراسية
- أساتذة يريدون مشاركة مواد
- مجموعات تعلّم ذاتي

### الميزات الخاصة:

| الميزة | الوصف | الأولوية |
|--------|-------|----------|
| تصنيف تلقائي | عند الرفع: محاضرة / ملخص / واجب / امتحان / مرجع | 🔴 |
| جدول مواعيد | تواريخ تسليم، امتحانات، محاضرات | 🟡 |
| تقييم الملفات | ⭐ هل هذا الملخص مفيد؟ (upvote/downvote) | 🟡 |
| أسئلة وأجوبة | سؤال مرتبط بملف أو مادة (mini Q&A) | 🟢 |
| تلخيص AI | تلخيص PDF/ملاحظات تلقائياً | 🟢 |
| إحصائيات المادة | كم ملف، كم عضو، آخر نشاط | 🟡 |
| قالب مساحة جاهز | "مادة جامعية" → مجلدات جاهزة (محاضرات/ملخصات/واجبات) | 🔴 |

### تجربة المستخدم:
```
مساحة: الرياضيات 1
├── 📂 محاضرات
│   ├── محاضرة 1 - المتتاليات.pdf  ⭐4.2
│   ├── محاضرة 2 - التكامل.pdf     ⭐3.8
│   └── ...
├── 📂 ملخصات
│   ├── ملخص الفصل الأول.pdf       ⭐4.9 🔥
│   └── ...
├── 📂 واجبات
│   ├── واجب 1 (تسليم: 15/6)       ⏰
│   └── ...
├── 📂 امتحانات سابقة
│   └── نهائي 2024.pdf
└── 📋 جدول المادة
```

## 4. المسار الفرقي (Teams Track) 👥

### الجمهور المستهدف:
- فرق عمل صغيرة (5-25 شخص)
- وكالات تصميم / تسويق
- شركات ناشئة
- فريلانسرز مع عملاء

### الميزات الخاصة:

| الميزة | الوصف | الأولوية |
|--------|-------|----------|
| مهام مرتبطة بملفات | "راجع هذا الملف" → مهمة لشخص معين | 🔴 |
| حالات الملف | Draft → In Review → Approved → Final | 🔴 |
| Mentions | @اسم في التعليقات → إشعار مباشر | 🟡 |
| قوالب مساحات | "مشروع تصميم" / "حملة تسويق" → مجلدات جاهزة | 🟡 |
| تقرير أسبوعي | ملخص نشاط الأسبوع لكل عضو | 🟢 |
| تكامل خارجي | Webhook / Zapier (لاحقاً) | 🟢 |
| عرض Kanban | ملفات كبطاقات حسب الحالة | 🟡 |

### تجربة المستخدم:
```
مساحة: إعادة تصميم الموقع
├── 📂 التصاميم
│   ├── Homepage_v3.fig    [✅ Approved]  → مهمة: تسليم للمطور
│   ├── About_v1.fig      [🔄 In Review] → مهمة: مراجعة أحمد
│   └── Contact_draft.fig [📝 Draft]
├── 📂 المحتوى
│   ├── نصوص الصفحة الرئيسية.docx [✅ Approved]
│   └── ...
├── 📂 الأصول (Assets)
│   ├── logo_final.svg
│   └── brand_colors.pdf
└── 📋 المهام المفتوحة (3)
```

## 5. المسار الآمن (Secure Track) 🔒

### الجمهور المستهدف:
- مكاتب محاماة
- مكاتب محاسبة
- عيادات ومراكز طبية
- استشاريون يتعاملون مع بيانات حساسة

### الميزات الخاصة:

| الميزة | الوصف | الأولوية |
|--------|-------|----------|
| انتهاء صلاحية الملف | الملف يصبح غير متاح بعد تاريخ معين | 🔴 |
| تتبع المشاهدات | من فتح الملف، متى، كم مرة | 🔴 |
| Watermark تلقائي | اسم المشاهد + التاريخ على PDF/صور | 🟡 |
| منع التحميل | مشاهدة فقط بدون تنزيل (لبعض الأدوار) | 🔴 |
| رمز وصول إضافي | PIN code للملفات الحساسة جداً | 🟡 |
| سجل وصول مفصّل | تقرير كامل: من دخل، ماذا شاهد، متى خرج | 🔴 |
| مشاركة خارجية محكومة | رابط لعميل خارجي بدون حساب (view-only + expires) | 🟡 |
| تشفير محسّن | تشفير إضافي على مستوى الملف | 🟢 |

### تجربة المستخدم:
```
مساحة: ملفات العميل — شركة XYZ
├── 📂 العقود
│   ├── عقد_التأسيس.pdf      🔒 [PIN] [expires: 30/7] [views: 3]
│   └── ملحق_1.pdf           🔒 [view-only] [watermarked]
├── 📂 التقارير المالية
│   ├── Q1_2026.pdf           🔒 [expires: 15/8]
│   └── ...
├── 📂 المراسلات
│   └── ...
└── 📊 سجل الوصول
    └── "أحمد شاهد عقد_التأسيس.pdf — 28/5 الساعة 14:32"
```

## 6. نظام الاشتراكات (Pricing Model) 💰

### الخطة المقترحة:

| الخطة | السعر | التخزين | المساحات | الميزات |
|-------|-------|---------|----------|---------|
| **Free** | $0 | 500 MB | 3 مساحات | النواة فقط + مسار واحد |
| **Pro** | $7/شهر | 10 GB | غير محدود | كل المسارات + AI features |
| **Team** | $5/عضو/شهر | 50 GB مشترك | غير محدود | Pro + إدارة فريق + analytics |
| **Enterprise** | تواصل | مخصص | مخصص | Team + SSO + SLA + API |

### ما الذي يدفع المستخدم للترقية؟
1. **حد التخزين** — 500 MB تنتهي بسرعة مع الملفات
2. **عدد المساحات** — 3 مساحات لا تكفي لطالب عنده 6 مواد
3. **ميزات المسار** — التصنيف التلقائي، المهام، التتبع الآمن
4. **AI** — التلخيص والبحث الذكي

## 8. قاعدة البيانات — التوسعات

### Collections الجديدة (إضافة على الموجود):

```
# النواة (موجود)
workspaces/{id}
workspaces/{id}/members/{uid}
workspaces/{id}/files/{fileId}
workspaces/{id}/file_versions/{versionId}
workspaces/{id}/file_comments/{commentId}
workspaces/{id}/audit_logs/{logId}
users/{uid}
users/{uid}/notifications/{notifId}

# جديد — مشترك
workspaces/{id}/folders/{folderId}
workspaces/{id}/tags/{tagId}

# جديد — المسار الأكاديمي
workspaces/{id}/schedule/{eventId}          ← مواعيد
workspaces/{id}/files/{fileId}/ratings      ← تقييمات

# جديد — المسار الفرقي
workspaces/{id}/tasks/{taskId}              ← مهام
workspaces/{id}/files/{fileId}/status_log   ← تاريخ تغيير الحالة

# جديد — المسار الآمن
workspaces/{id}/files/{fileId}/access_log   ← سجل مشاهدات
workspaces/{id}/external_shares/{shareId}   ← روابط خارجية
```

### تعديل على Workspace document:
```typescript
interface Workspace {
  // ... الحقول الموجودة
  track: 'general' | 'edu' | 'teams' | 'secure';  // ← جديد
  track_config?: {                                  // ← جديد
    edu?: { auto_classify: boolean; ratings_enabled: boolean };
    teams?: { statuses: string[]; tasks_enabled: boolean };
    secure?: { watermark: boolean; access_tracking: boolean };
  };
}
```

## 9. خارطة الطريق (Roadmap) — 4 مراحل

### المرحلة 1: تقوية النواة (4-6 أسابيع) 🔴

**الهدف:** جعل الأساس صلباً قبل إضافة المسارات

| المهمة | التفاصيل | الأسبوع |
|--------|----------|---------|
| نظام مجلدات | إنشاء/نقل/حذف مجلدات + breadcrumb | 1-2 |
| بحث متقدم | فلترة حسب النوع + الحجم + التاريخ + المجلد | 2-3 |
| Tags | إضافة/إزالة tags + فلترة بها | 3 |
| Trash bin | حذف ناعم + استعادة خلال 30 يوم | 3-4 |
| اختيار نوع المساحة | UI لاختيار track عند الإنشاء | 4 |
| هيكلة الكود | نقل الكود لبنية core/tracks | 4-5 |
| تحسين الأداء | Pagination حقيقي + caching | 5-6 |

### المرحلة 2: المسار الأكاديمي (4-5 أسابيع) 📚

**الهدف:** إطلاق أول مسار متخصص — الأكثر طلباً

| المهمة | التفاصيل | الأسبوع |
|--------|----------|---------|
| قالب مساحة أكاديمية | مجلدات جاهزة عند الإنشاء | 1 |
| تصنيف الملفات | اختيار يدوي + اقتراح تلقائي بالاسم | 1-2 |
| تقييم الملفات | ⭐ rating + ترتيب حسب التقييم | 2-3 |
| جدول المادة | إضافة مواعيد + تذكيرات | 3-4 |
| إحصائيات المادة | عدد الملفات، أكثر ملف مشاهدة، إلخ | 4-5 |

### المرحلة 3: المسار الفرقي (4-5 أسابيع) 👥

**الهدف:** جذب فرق العمل الصغيرة

| المهمة | التفاصيل | الأسبوع |
|--------|----------|---------|
| حالات الملف | Draft/Review/Approved/Final + UI | 1-2 |
| نظام مهام بسيط | مهمة مرتبطة بملف + assign + due date | 2-3 |
| Mentions | @اسم في التعليقات | 3 |
| قوالب مساحات فرقية | "مشروع تصميم" / "حملة" | 3-4 |
| عرض Kanban | ملفات كبطاقات حسب الحالة | 4-5 |

### المرحلة 4: المسار الآمن + الاشتراكات (5-6 أسابيع) 🔒💰

**الهدف:** تحقيق إيرادات + جذب القطاع المهني

| المهمة | التفاصيل | الأسبوع |
|--------|----------|---------|
| تتبع المشاهدات | سجل: من فتح، متى، مدة | 1-2 |
| انتهاء صلاحية الملف | تاريخ + إخفاء تلقائي | 2 |
| منع التحميل | view-only mode لأدوار معينة | 2-3 |
| Watermark | اسم المشاهد + تاريخ على PDF | 3-4 |
| مشاركة خارجية | رابط لغير المسجلين (محكوم) | 4 |
| نظام اشتراكات | Stripe + خطط + حدود | 4-6 |
| صفحة Pricing | عرض الخطط + مقارنة | 6 |
